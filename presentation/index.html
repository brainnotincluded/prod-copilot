ФИЧА: КАРТА API И СУЩНОСТЕЙ

Что видит пользователь

  • ▸ Все подключённые API и их возможности
  • ▸ Какие действия доступны именно ему (по роли)
  • ▸ Read / Write разграничение
  • ▸ Визуальная схема связей между сущностями
  • ▸ Подсказки по комбинированию блоков

Зачем это важно

Пользователь видит реальные границы возможного до старта сценария — система не обещает невозможного и не создаёт ложных ожиданий

Новые capabilities подключаются декларативно или через ограниченное расширение кода

Vue Flow визуализация — группировка по API, FK-связи, легенда методов

SWOT

Strengths · Сильные стороны

  • Универсальность поверх любых API — не привязан к домену
  • Human-in-the-loop подтверждения для state-changing шагов
  • Авто-генерируемые рабочие интерфейсы — уникально vs Zapier/n8n
  • Полная прозрачность шагов и reasoning модели
  • Декларативное подключение новых capabilities

Weaknesses · Слабые стороны

  • Требует качественного описания API для корректной работы ML
  • Сложность UX при большом числе capabilities
  • Зависимость качества от LLM — нужен fallback
  • Cold-start: нет истории запусков при первом запуске

Opportunities · Возможности

  • Уход западных вендоров — Microsoft Copilot Studio недоступен в РФ
  • Взрывной рост agentic AI: CAGR 43.84% до 2030
  • Растущий спрос на внутреннюю автоматизацию в корпорациях
  • Шаблоны сценариев — переиспользование и быстрый onboarding

Threats · Угрозы

  • Microsoft Copilot Studio — нативный multi-agent orchestration
  • Собственные AI-платформы крупных игроков (Яндекс, Сбер)
  • Галлюцинации LLM → риск выполнения некорректных шагов
  • Сопротивление adoption: команды привыкли к ручному контролю

E2E ДЕМО: МАРКЕТИНГОВАЯ ПЛАТФОРМА

Сценарий

Маркетолог описывает задачу в чате → система собирает E2E маршрут из 6 шагов

Шаги сценария

1Поиск сегментов: Премиум + Неактивные 60+ + Высокий чек
2Прогноз аудитории для разных логик пересечения
3[!]Создание аудитории · approval
4Генерация 2–3 вариантов push-уведомления
5[!]Создание черновика кампании · approval
6Авто-интерфейс: KPI, график, карточки, журнал

Что проверяет демо

—Подключённые API и доступные действия
—Карта блоков и связей между ними
—Чат → план → уточнения
—Approval от владельца API
—Исполнение + рабочий интерфейс результата
—Drill-down в сегмент / шаг / контент
—Подключение новой capability (расширяемость)

Домен — пример, не ограничение.
Система работает для любой платформы с описанными API
